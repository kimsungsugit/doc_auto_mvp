# 공개 샘플 소스 정리

## 목적

세금계산서 자동입력 MVP의 샘플 확보 단계에서 사용할 공식/공개 자료 출처를 정리한다.  
이 문서는 실제 업무용 비식별 샘플이 확보되기 전까지 다음 목적에 사용한다.

1. 전자세금계산서 필드 구조 확인
2. 세금계산서 용어 및 작성 형식 확인
3. 테스트용 샘플 PDF 생성 기준 확보
4. 이후 실제 비식별 샘플 보정 시 기준 자료로 활용

## 1. 공식 참고 문서

### 국세청 전자세금계산서 발급방법 및 발급절차
- 링크: https://b.nts.go.kr/nts/cm/cntnts/cntntsView.do?cntntsId=7788&mi=2462
- 용도:
  1. 전자세금계산서 제도 및 발급 절차 이해
  2. 필수 기재 항목 구조 확인
  3. 홈택스 기반 발급 흐름 이해

### 홈택스 수정전자세금계산서 발급 안내 PDF
- 링크: https://teet.hometax.go.kr/doc/et/b/a/b/%EC%A1%B0%ED%9A%8C%EB%B0%9C%EA%B8%89_%EC%A0%84%EC%9E%90%EC%84%B8%EA%B8%88%EA%B3%84%EC%82%B0%EC%84%9C_%EC%88%98%EC%A0%95%EC%A0%84%EC%9E%90%EC%84%B8%EA%B8%88%EA%B3%84%EC%82%B0%EC%84%9C.pdf
- 용도:
  1. 수정 발급 시 필드 구조와 화면 흐름 참고
  2. 문서 용어와 상태값 참고
  3. OCR/추출 대상 라벨 후보 보강

## 2. 공개 양식 참고 소스

### 공폼 무료양식 메인
- 링크: https://gongform.com/
- 검색 결과 기준 확인 내용:
  1. 세금계산서 양식 PDF/HWP/XLS 제공 안내
  2. 세금계산서 양식 다운로드용 공개 양식 진입점으로 활용 가능
- 용도:
  1. 테스트용 레이아웃 다양성 확보
  2. 비공식 양식 구조 확인

## 3. 수집 원칙

공개 자료는 아래 기준으로만 사용한다.

1. 공식 문서:
   제도, 용어, 라벨, 작성 흐름 참고
2. 공개 양식:
   레이아웃 패턴 참고
3. 실제 추출 정확도 검증:
   반드시 비식별 실문서 또는 사용자 제공 샘플로 추가 보정

## 4. 현재 프로젝트 반영 방식

현재 프로젝트는 공개 자료를 직접 배포 자산으로 포함하지 않고, 아래 방식으로 반영한다.

1. `samples/sample_cases.json`
   공식/공개 자료에서 확인한 필드 구조를 바탕으로 테스트 케이스 정의
2. `scripts/generate_sample_pdfs.py`
   테스트용 PDF 10건 생성
3. `tests/test_extractor.py`
   레이아웃 변형 회귀 테스트
4. `tests/test_sample_cases.py`
   샘플 정의 무결성 검사

## 5. 다음 작업 연결

이 단계 다음 우선순위는 아래와 같다.

1. 비식별 실제 세금계산서 PDF 2~3건 확보
2. 공개 샘플과 실제 샘플의 필드 차이 비교
3. OCR 결과 기준 라벨 사전 보강
4. 실엑셀 양식 매핑 반영
5. OpenAI 구조화 계층 연결

## 6. 한계

1. 공개 웹에서 바로 사용할 수 있는 실제 전자세금계산서 PDF 샘플은 매우 제한적이다.
2. 공식 문서는 대부분 제도 안내 중심이고, 실무 문서 편차를 충분히 제공하지 않는다.
3. 따라서 현재 공개 소스 단계는 “샘플 생성 기준 확보” 목적에 가깝고, 최종 정확도 보정은 실제 비식별 샘플이 필요하다.
