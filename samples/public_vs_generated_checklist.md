# 공개 샘플 vs 생성 샘플 비교 체크리스트

## 목적

`public_sample_sources.md`를 기준으로 공개 자료에서 확인한 필드/레이아웃 특성과 현재 프로젝트에서 생성한 테스트용 샘플 PDF의 차이를 비교한다.  
이 체크리스트는 실제 비식별 샘플 PDF가 들어오기 전까지 추출기 보정 우선순위를 정하는 기준으로 사용한다.

## 1. 문서 메타 구조

- [x] 문서 제목에 `세금계산서` 또는 `전자세금계산서`가 포함되는가
- [x] 작성일자 라벨이 `작성일자`, `작성 일자`, `작성년월일` 변형으로 들어오는가
- [ ] 승인번호, 전송일시, 발급유형 등 전자문서 메타 필드가 포함되는가
- [ ] 공급시기(월/일) 필드가 별도 열로 분리되는가

## 2. 거래 당사자 정보

- [x] 공급자 사업자번호가 포함되는가
- [x] 공급받는자 사업자번호가 포함되는가
- [x] 공급자/공급받는자명이 같은 줄 또는 인접 줄로 표현되는가
- [ ] 대표자명, 주소, 업태, 종목이 별도 필드로 포함되는가
- [ ] 이메일, 담당자 정보가 포함되는가

## 3. 금액 정보

- [x] 공급가액, 세액, 합계가 추출 가능한 형태인가
- [x] `합계`, `총액`, `청구금액` 변형 라벨을 지원하는가
- [x] 쉼표 포함 숫자 정규화가 가능한가
- [ ] 음수 또는 수정세금계산서 금액 구조를 지원하는가
- [ ] 면세/영세율 구분 필드를 별도로 반영하는가

## 4. 품목/행 단위 구조

- [x] `품목`, `품명` 단일 항목 추출이 가능한가
- [ ] 다중 품목 행을 개별 라인 아이템으로 분리할 수 있는가
- [ ] 규격, 수량, 단가, 공급가액이 표 구조로 제공될 때 행 단위 파싱이 가능한가
- [ ] 비고가 여러 줄일 때 누락 없이 추출 가능한가

## 5. 레이아웃 편차

- [x] 한 줄 인라인 배치
- [x] 콜론 기반 배치
- [x] 금액이 한 줄에 밀집된 배치
- [x] 공백이 불규칙한 배치
- [ ] 스캔/캡처형 PDF 배치
- [ ] 표/박스 중심 레이아웃
- [ ] 공급자/공급받는자 위치가 좌우 컬럼으로 나뉜 레이아웃

## 6. OCR 관점

- [x] 텍스트 PDF 기준 추출 테스트가 존재하는가
- [x] OCR 필요 문서 경로가 구현되어 있는가
- [ ] 실제 스캔 PDF로 OCR 결과를 검증했는가
- [ ] OCR 결과에서 자주 깨지는 한글 라벨 사전을 보강했는가

## 7. 현재 생성 샘플이 커버하는 범위

현재 `samples/generated/*.pdf`는 아래 범위를 커버한다.

1. 기본 인라인형
2. 콜론형
3. 금액 밀집형
4. 공백 변형형
5. 긴 품목명
6. 일부 필드 누락형
7. 고액 금액형
8. 비고 강조형
9. 압축된 사업자번호/상호 표현형
10. 서비스 품목형
